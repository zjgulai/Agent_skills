Idea → PR · the full chain

A single feature trip. Each box is a real tool invocation with evidence in docs/demo/E2E-DEMO.md.

  1. Idea forms — frontend-design skill loads
    From anthropics/skills via real git clone. Agent picks a visual direction instead of the SaaS-template mean.
  2. Page generated — agent-kit.html
    This page right here. Serif headlines, amber accent, glass panels, count-ups.
  3. Render check — playwright MCP (or fallback)
    If playwright MCP is installed, the agent navigates and screenshots; if not, falls back to open.
  4. Self review — code-review skill
    Semantic HTML, AA contrast, fallback fonts, no inline JS, no surprise dependencies.
  5. Cleanup attempt — guard-bash hook blocks
    Agent tries rm -rf .sisyphus/snapshots → hook returns exit 2 → opencode shows reason to user.
  6. PR drafted — create-pr skill
    Title, summary, verification, risks, rollback. Reviewer-ready.
  7. Stop — final-verify hook reports
    Changed files, open TODOs, optional test run, rendered as a JSON summary the agent surfaces.
