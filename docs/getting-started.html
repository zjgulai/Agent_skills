Getting Started

From a fresh machine to a working Skills Manager in roughly 10 minutes.

Prerequisites

ToolMin versionCheck
git2.30+git --version
Python3.9+python3 --version
Node22+node --version
npm11+npm --version
OpenCode CLIlogged inopencode --version

Optional but recommended on macOS: portless for https://*.localhost.

1. Clone and bootstrap the portal

Clone wherever you keep your projects. The path is up to you; we'll use ~/project/Agent as an example.

mkdir -p ~/project/Agent
cd ~/project/Agent
git clone https://github.com/zjgulai/Agent_skills.git
cd Agent_skills

Install the portal backend (FastAPI in an isolated venv).

cd portal/backend
python3 -m venv .venv
.venv/bin/pip install --upgrade pip
.venv/bin/pip install fastapi 'uvicorn[standard]' pyyaml python-multipart httpx mistune pytest
cd ../..

Install the portal frontend dependencies.

cd portal/frontend
npm install
cd ../..

Smoke-test the portal.

./portal/bin/start
# Look for "✅ Skills Portal up:" then Ctrl+C

2. Bootstrap the skills repo

The skill repo lives at ~/.config/opencode/skills/ (this is OpenCode's convention; do not relocate). If you've never used OpenCode skills before, create the bare scaffold:

mkdir -p ~/.config/opencode/skills
cd ~/.config/opencode/skills
touch INDEX.md skills-graph.mmd

Seed INDEX.md with the 6-domain skeleton (an example is in docs/INSTALL.md). Seed skills-graph.mmd with a minimal mermaid header.

If you already have a working skills repo, skip this step.

3. Wire OpenCode

OpenCode auto-discovers commands and subagents. Just open the repo:

cd ~/project/Agent/Agent_skills
opencode

Type /skill- in the chat — you should see all 11 commands autocomplete.

4. First run

The end-to-end smoke flow:

/portal-start
/skill-list
/skill-doctor
/skill-install https://github.com/op7418/guizang-ppt-skill

If /skill-install finishes with all green ✅ checks and the portal UI shows the new skill in the right domain, you're done.

Verification

Run the test suite to confirm everything is wired correctly:

portal/backend/.venv/bin/python -m pytest tests/ -q
# Expected: 20 passed

Troubleshooting

Ports 5173 / 5174 already in use

lsof -ti:5173,5174 | xargs kill -9

Browser shows "400" or fails to load portal

System HTTP proxy is intercepting localhost. Check macOS System Settings → Network → Proxies, or set bypass for localhost,127.0.0.1,*.localhost.

Graph rendering errors

Install playwright + chromium for render-mermaid.py:

~/Library/Python/3.9/bin/pip install playwright
~/Library/Python/3.9/bin/python -m playwright install chromium
