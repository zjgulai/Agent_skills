  • Read-only contexts where you can't run commands (airplane, client laptop, pure web)

§2. Domain Taxonomy

Six domains. One rule: when the classifier is uncertain, ask the user. The classifier never silently miscategorizes.

How inference works

  1. Concatenate name + description, lowercase
  2. For each domain, count how many of its keywords appear (each keyword counted at most once)
  3. Pick the highest-scoring domain. Ties broken by priority order:
ip → desktop → founder → meta → closeout → tooling
  1. If every score is 0 → returns uncategorized; the agent prompts the user to manually pick

The 6 domains

meta AI Engineering Infrastructure

When to enter: the task targets AI agents themselves — repository scaffolding, agent memory, skill libraries, hooks, subagents, plugins, agent self-governance.

Note: bare skill is NOT a meta keyword — too generic; almost every SKILL.md mentions it. Only subagent, agent, and explicit config files are strong signals.

closeout Code Quality & Delivery Closeout

When to enter: code is written, you're at the last mile — commit / PR / release / lint / audit.

Updating the taxonomy

When updating keywords: edit agent/docs/domain-taxonomy.md first, then mirror the change in agent/lib/domain_inference.py KEYWORDS table, finally run pytest regression.

§5. Graph Case Studies

4 graph states walking through how the graph mutates as new skills are added.

5.1 State 0: initial 6 skills

The very first state: 6 user skills + 5 builtins. One main lifecycle chain founder → meta → desktop → closeout → ip.

  • 6 user skills: ADK, CR, NF, SPT, PAT, SCS
  • 5 builtins (dashed): GM, RW, ASR, FE, PW
  • Main chain: SPT → ADK → NF → CR → PAT → SCS

5.2 State 1: after installing guizang-ppt-skill

Triggered by: /skill-install https://github.com/op7418/guizang-ppt-skill

tooling domain block is auto-created; GPS joins as isolated node. Agent adds toolingDomain classDef.

Δ vs previous state: +1 node (GPS in tooling) · +1 classDef · +1 self-loop edge · PNG 224500 → 259330 (+15.5%)
  • Inference: tooling (matched ppt / swiss style / html, score 3)
  • New node: GPS[guizang-ppt-skill]:::toolingDomain
  • New edge: GPS self-loop (no same-domain peer yet)

5.3 State 2: after installing skill-creator

Triggered by: /skill-install https://github.com/anthropics/skills --subdir skills/skill-creator

Description too generic; inference returns uncategorized; user manually picks meta. SC joins meta with ADK; dashed same-domain edge added.

Δ vs previous state: +1 node (SC in meta) · +1 same-domain edge → ADK · PNG 259330 → 270529 (+4.3%)
  • Inference: uncategorized → user picks meta manually
  • New node: SC[skill-creator]:::metaDomain
  • New edge: SC -. "same domain" .-> ADK

5.4 State 3: after installing agent-reach

Triggered by: pipx install agent-reach && agent-reach install --env=auto

agent-reach is a special skill — it ships as a Python CLI (pipx install agent-reach) and writes SKILL.md to ~/.agents/skills/. Symlinked into opencode and portal. Classified as tooling, peer to GPS.

Δ vs previous state: +1 node (AR in tooling) · +1 same-domain edge → GPS · PNG 270529 → 280641 (+3.7%)
  • Inference: meta (matched 'agent') → manually reclassified to tooling (better fits cross-cutting)
  • New node: AR[agent-reach]:::toolingDomain
  • New edge: AR -. "same domain" .-> GPS (tooling now has a peer)
  • Bridge: ~/.config/opencode/skills/agent-reach → ~/.agents/skills/agent-reach (symlink)
