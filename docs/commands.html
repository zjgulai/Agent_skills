Commands Reference

11 slash commands organized by lifecycle phase.

Skill lifecycle

/skill-install <url> [--subdir X]

write

Install a skill from a GitHub URL. The most exercised command.

7-step orchestration

  1. portal_client.ensure_running() — bring portal up if down
  2. POST /api/install/github {url, subdir?} — clone + validate + copy
  3. GET /api/skills/<name> — read frontmatter
  4. domain_inference.infer(name, description) — classify by keywords
  5. index_md_writer.append(name, domain, role, triggers) — INDEX.md row
  6. graph_writer.add_node(name, domain) + render() — graph + PNG
  7. POST /api/refresh — refresh portal index

If domain inference returns uncategorized, the agent prompts the user to pick one of 6 domains manually.

Example

/skill-install https://github.com/op7418/guizang-ppt-skill
/skill-install https://github.com/anthropics/skills --subdir skills/skill-creator

/skill-uninstall <name>

write

Reverse of install. Idempotent metadata cleanup.

4-step cleanup

  1. Confirm skill exists, ask user for explicit "y" confirmation
  2. graph_writer.remove_node() + render — graph first to avoid orphan
  3. index_md_writer.remove() — INDEX.md row removal, restore > 暂为空 placeholder if domain becomes empty
  4. DELETE /api/skills/<name> + refresh — physical files

/skill-update <name | all>

write

git pull --ff-only in skill directories, then refresh portal.

  • Per-skill outcome reported: Already up to date / Updated (N commits) / ERROR: merge conflict
  • Does not touch INDEX.md / graph — content updates only. Run /skill-classify if a pull suggests a domain change.

Inspection

/skill-list

read

Print all installed skills, grouped by domain, with a one-line description and top triggers.

/skill-recommend <task description>

read

Given a free-form task, recommend a load_skills=[...] combo.

Strategy

  1. Match against INDEX.md "典型场景套餐表" first (curated combos)
  2. If no scenario hit, score each skill against task keywords; recommend top 1-3
  3. Output a copy-paste-able task(category=..., load_skills=[...], ...) template

Example

/skill-recommend "做一份瑞士风 PPT"

/skill-doctor [name | all]

read

Health check across 5 rules. No arg → check all skills.

5 rules

  • R1 — SKILL.md frontmatter parses (name + description)
  • R2 — directory name matches frontmatter name
  • R3 — present in INDEX.md
  • R4 — present in skills-graph.mmd with matching domain
  • R5 — declared external dependencies are available

Outputs PASS / WARN / FAIL per skill. R5 only warns (skills can run in degraded mode); R1-R4 fail blocks.

Maintenance

/skill-classify <name>

write

Manually move a skill to a different domain. Use when /skill-install picked the wrong one or when keywords drift over time.

Lists the 6 domain options, waits for user to pick one, then moves the row in INDEX.md and updates the graph node + classDef.

/skill-graph-sync

write

Reconcile skills-graph.mmd against INDEX.md (truth source). Idempotent.

Detects three drift categories — MISSING / EXTRA / MISMATCH — and offers to fix all three. Re-renders PNG only if there were changes.

Portal control

/portal-start

lifecycle

Bring the portal online. Idempotent (no-op if already running).

  • Pre-flight: venv exists, ports 5173/5174 free
  • Spawns portal/bin/start, redirects logs to /tmp/portal-uvicorn.log
  • Polls /api/health for up to 20 seconds
  • Auto-detects portless for HTTPS via https://skills-portal.localhost

/portal-stop

lifecycle

Forcefully kill processes on ports 5173 + 5174. Idempotent.

Equivalent to lsof -ti:5173,5174 | xargs kill -9.

/portal-status

read

Diagnostic snapshot: API up?, port states, PIDs, last index time, skill count.
