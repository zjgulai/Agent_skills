Domain Taxonomy

Six domains. One rule: when uncertain, ask the user. The classifier never silently miscategorizes.

How inference works

Every skill has a name + description in its SKILL.md frontmatter. domain_inference.infer(name, description) does:

  1. Lowercase the concatenation of name + description
  2. For each domain, count how many of its keywords appear (each keyword counts at most once)
  3. Pick the highest-scoring domain. Ties resolved by domain priority: ip → desktop → founder → meta → closeout → tooling
  4. If all scores are 0 → return uncategorized. The agent then prompts the user.

Regression-tested 7/7 against the 7 currently-installed skills.

The six domains

meta

AI 工程基础设施

When to enter:

The task targets AI agents themselves: repository scaffolding, agent memory, skill libraries, hooks, subagents, plugins, agent self-governance.

Strong-signal keywords:

Note: bare skill is NOT in the meta keyword list — it's too generic; almost every SKILL.md description mentions "skill". Only subagent, agent, and explicit config files are strong signals.

兜底

Returned when zero domain keywords match. The agent prompts the user to manually pick one of the 6 domains. /skill-classify can fix this later.

Updating the taxonomy

  1. Edit agent/docs/domain-taxonomy.md first — it's the human-readable truth source
  2. Mirror the changes in agent/lib/domain_inference.py KEYWORDS table
  3. Run pytest tests/test_domain_inference.py -v to ensure regression cases still pass
  4. If a previously-installed skill now infers differently, run /skill-classify <name> to migrate
