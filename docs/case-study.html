Case Studies

Two real installs that exercised the full lifecycle, including the edge cases.

Case 1: guizang-ppt-skill

Path: auto-classify happy path. From URL to a fully-indexed, doctor-PASS skill.

Trigger

/skill-install https://github.com/op7418/guizang-ppt-skill

What happened

Step 1 — portal ensure

✅ Portal API http://127.0.0.1:5174/api/health returned {ok: true}.

Step 2 — install via portal

✅ Portal's installer.py ran:

  • git clone --depth 1 to a tmp dir
  • Found SKILL.md at repo root (no --subdir needed)
  • Validated frontmatter: name: guizang-ppt-skill matches ^[a-z0-9]+(-[a-z0-9]+)*$
  • Copied to ~/.config/opencode/skills/guizang-ppt-skill/ (skipping .git, LICENSE, README.md)

Step 3 — domain inference

Description contains "polished HTML slide decks: editorial magazine and Swiss layouts, image prompts, social covers".

✅ Auto-classified as tooling with 3 keyword hits.

Step 4 — INDEX.md row

index_md_writer.append("guizang-ppt-skill", "tooling", ...)

  • Atomic write via os.replace — INDEX.md grew 16050 → 20148 bytes
  • Post-write verification: re-parse confirmed guizang-ppt-skill in tooling. Pass.

Case 2: skill-creator

Path: manual-fallback path. Tests the uncategorized → user picks domain branch and the --subdir flag for monorepos.

Trigger

/skill-install https://github.com/anthropics/skills --subdir skills/skill-creator

What was different

Subdir resolution

The official anthropic skills repo is a monorepo with 17 SKILL.md files. Without --subdir, the installer would error out asking which one to pick. With --subdir skills/skill-creator, the installer scoped its search and found exactly one SKILL.md.

Domain inference returned uncategorized

The skill description: "Create new skills, modify and improve existing skills, and measure skill performance..."

You'd think this would auto-classify as meta. But the keyword skill alone is intentionally NOT in the meta keyword list (too generic — every SKILL.md mentions it). Without other meta signals (agent, subagent, hook, plugin), the score is 0 across all domains:

{
  "domain": "uncategorized",
  "scores": {},
  "evidence": {}
}

Manual classification

The agent prompted the user to pick from 6 domains. Picked meta manually (skill that creates skills = meta-engineering).

Rest of the flow

Identical to Case 1 from this point: append to INDEX.md (under existing meta domain, this time appending to a non-empty table), add graph node SC, render PNG (270529 bytes), refresh portal (7 → 8 skills).

Doctor

$ /skill-doctor skill-creator
overall: PASS  (5/5 rules)

What these cases proved

  • The happy path is fast. ~30 seconds from URL to fully-indexed skill.
  • Monorepo subdir works. The portal's --subdir flag handled skills/skill-creator without issue.
  • The "uncategorized" branch is real, not aspirational. The classifier deliberately refuses to guess when keyword evidence is weak. Better to ask once than miscategorize forever.
  • All three meta files stay coherent. INDEX, graph, and PNG always agree because the agent updates them in a single transactional flow with rollback on verification failure.
